# 테스트 주도 개발(Test Driven Development)
- 실제 코드를 작성하기 전에 테스트 코드 먼저 작성
- 테스트 코드 작성 후 테스트 코드를 pass할 수 있는 실제 코드 작성
	1. 원하고자 하는 기능의 테스트 코드 작성
	2. 테스트 실행 (Fail)
	3. 테스트 코드에 맞는 실제 코드 작성
	4. 테스트 실행 (Pass)
## TDD를 하면 좋은 점
1. 많은 기능을 테스트하기에 소스 코드에 안정감이 부여됨
2. 디버깅 시간이 줄어들고 실제 개발 시간도 줄어듦
3. 소스 코드 하나하나를 더욱 신중하게 짤 수 있기 때문에 깨끗한 코드가 나올 확률이 높음
