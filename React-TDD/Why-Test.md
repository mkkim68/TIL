# 테스팅의 이유
## 왜 어플리케이션을 Test 해야 할까요?
- 간단하게 더 안정적인 어플리케이션을 위해서는 여러 방법으로 테스트를 해줘야 더 안정적인 어플리케이션이 될 수 있음
## 테스팅으로 얻는 이점
### 1. 디버깅 시간 단축
- 만약 데이터가 잘못 나왔다면 그것이 UI의 문제인지 DB의 문제인지 등을 전부 테스트 해봐서 찾아야 함
- 하지만 테스팅 환경이 구축되어있다면 자동화된 유닛 테스팅으로 특정 버그를 쉽게 찾아낼 수 있음
### 2. 더욱 안정적인 어플리케이션
- 많은 테스트 코드와 함께 작성된 코드의 어플리케이션이 되기 때문에 훨씬 안정적
### 3. 기타
- 재설계 시간 단축
- 추가 기능 구현 시 용이함
